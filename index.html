moip-assinaturas.js


O moip-assinaturas.js é uma biblioteca JavaScript feita para facilitar a integração do Moip Assinaturas com o seu site, permitindo o desenvolvimento do seu próprio fluxo de contratação e que seu cliente consiga assinar um plano sem sair do seu site, assim como funciona o CheckOut Transparente do Moip. Além de possuir uma estrutura e validações pré-definidas, utilizando essa biblioteca você também reduz o seu escopo para o PCI, uma vez que os dados de cartão de crédito vão direto do navegador do seu cliente para o Moip.

Configurando Obrigatório

O primeiro passo é incluir a tag script abaixo logo após o <head> para chamar o moip-assinaturas.js.

Exemplo:
<script src="https://api.moip.com.br/moip-assinaturas.js" type="text/javascript"></script>

Identificação Obrigatório para todos os serviços do moip-assinaturas.js

Para autenticar-se nas APIs do Moip Assinaturas é necessário informar seu token e key. Para a sua segurança, esses dados devem ser obrigatoriamente enviados por um Hash de Autorização. Este Hash deve gerado a partir da concatenação do token, key e de uma variável (salt) e, depois, encriptografado em SHA-1.

Importante: por segurança, também não recomendamos gerar o hash de autorização via JavaScript.

Abaixo, algumas implementações de como gerar um hash de autorização para a utilização dos serviços.

    public static String sha1(String input) throws NoSuchAlgorithmException {
        MessageDigest mDigest = MessageDigest.getInstance("SHA1");
        byte[] result = mDigest.digest(input.getBytes());
        StringBuffer sb = new StringBuffer();
        for (int i = 0; i < result.length; i++) {
            sb.append(Integer.toString((result[i] & 0xff) + 0x100, 16).substring(1));
        }
        return sb.toString();
    }
}

Criar uma assinatura

Pré-requisitos

  • Criação prévia de um plano. Saiba como criar um plano
  • Ter importado o javascript moip-assinaturas.js. Saiba mais
  • Ter gerado o Hash de Autorização. Saiba mais

Criando o cliente new Customer()

O assinatura necessita de um cliente, para criar um cliente bastar instanciar o objeto new Customer() conforme exemplo. O cliente possui suas informações pessoais (nome, e-mail, código do cliente, cpf, data de nascimento, etc), o endereço new Address()(rua, numero, bairro, etc) e o os dados de cobraça new BillingInfo()(cartão de crédito).


Criando a assinatura new MoipAssinaturas()

O primeiro passo para a utilização do moip-assinaturas.js é instanciar o objeto new MoipAssinaturas() essa classe javascript é responsável por realiazar a integração com o Moip Assinaturas. Em seu construtor ela recebe dois parâmetros um é o Hash de Autorização e o outro é o Salt (palavra chave utilizada para gerar o Hash de Autorização)

O metódo MoipAssinaturas.subscribe é responsavel por criar uma nova assinatura com um novo cliente. Ele recebe como parâmetro um objeto new Subscription() onde tem como parâmetros o código da assinatura, o novo cliente(criado atráves do objeto Customer) e o código do plano.

Abaixo há um exemplo de um HTML e seu respectivo javascript para gerar uma assinatura.

Criando a assinatura no Moip Assinaturas:
var moip = new MoipAssinaturas("IDENTIFICAÇÃO", "SEU_SALT");
var customer = build_customer();
var plan_code = $("#plan_code").val();

moip.subscribe(
    new Subscription()
        .with_code(new Date().getTime())
        .with_new_customer(customer)
        .with_plan_code(plan_code);
).callback(function(response){
    if (response.has_errors()) {
        for (i = 0; i < response.errors.length; i++) {
          var erro = response.errors[i].description;
          $("#erros").append(erro);
        }
        return;
    }
    alert("Assinatura criado com sucesso")
});
Criando o cliente a partir do formulário:
var build_customer = function() {
  var customer_params = {
      fullname: $("#fullname").val(),
      email: $("#email").val(),
      code: $("#fullname").val().toLowerCase()),
      cpf : $("#cpf").val(),
      birthdate_day : $("#birthdate_day").val(),
      birthdate_month: $("#birthdate_month").val(),
      birthdate_year: $("#birthdate_year").val(),
      phone_area_code: $("#ddd").val(),
      phone_number: $("#phone").val(),
      billing_info: build_billing_info(),
      address: build_address()
  }
  return new Customer(customer_params);
};

var build_billing_info = function() {
  var billing_info_params = {
      fullname : $("#holder_name").val(), 
      expiration_month: $("#expiration_month").val(),
      expiration_year: $("#expiration_year").val(),
      credit_card_number: $("#credit_card").val()
  };
  return new BillingInfo(billing_info_params);
};

var build_address = function() {
  var address_params = {
      street: $("#rua").val(),
      number: $("#numero").val(),
      complement: $("#complemento").val(),
      district: $("#bairro").val(),
      zipcode: $("#cep").val(),
      city: $("#cidade").val(),
      state: $("#estado").val(),
      country: "BRA"
  };
  return new Address(address_params);
};

Atualizar dados de cartão

Para atualizar os dados de cartão do um cliente é necessário construir um objeto new Customer() onde obrigatóriamente deve conter o código do cliente que você deseja atualizar os dados de cartão. E também obrigatóriamente deve conter a nova new BillingInfo() com os novos dados de cobrança.

/
Atualizando um Cartão de credito
$("#update_billinginfo").click(function(){
  moip = new MoipAssinaturas("IDENTIFICAÇÃO", "SEU_SALT");
  var customer = build_customer();
  moip.update_credit_card(customer).callback(function(data){
    if(data.has_errors()) {
      alert("Ops.... houveram erros");
    }
    alert("O cartão de credito foi atualizado")
  });
});

Callback personalizando o retorno do moip

Passando um método de callback

Assim que o Moip Assinaturas conclui o sua requisição o metódo de callback é executado. Isto é útil para você dar sequência no seu checkout ou mesmo exibir mensagens de erros do cliente ao preencher o checkout.

Para que possamos chamar seu metódo de callback é necessário que você passe o bloco de função para o metódo callback.

Exemplo:
var moip = new MoipAssinaturas("IDENTIFICAÇÃO", "SEU_SALT");
moip.callback(function(response){
  // aqui você adiciona os seus comportamentos esperados
});

Processando a resposta

O Moip Assinaturas irá chamar seu callback passando como parâmetro um objeto Response(), ele contem uma lista de possíveis erros ou por informações erradas preenchidas pelo seu usuário ou na integração. O Response também possui um lista de alertas, esses alertas auxiliam você na sua integração.

Para verificar se há erros ou alertas na resposta do Moip Assinaturas basta chamar o metódos has_errors() ou has_alerts().

Exemplo:
var moip = new MoipAssinaturas("IDENTIFICAÇÃO", "SEU_SALT");
moip.callback(function(response){
  alert(response.has_errors());
  alert(response.has_alerts());
});
